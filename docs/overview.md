# 오늘의집 3D 에셋 라이브러리 — 프로젝트 브리핑

> 참고: [26YQ1 오늘의집 시각 자산정리 및 체계화 플랜](https://www.notion.so/26YQ1-2e2a597878a08041af42f97d0a1171ba?pvs=21)

---

## 1. 배경 — 왜 새로 만드나

오늘의집은 현재 3D 에셋을 **오프라인 렌더(C4D 등) → 정적 이미지** 방식으로 운영하고 있고, 다음 한계가 누적되어 있음.

- **각도·컬러 변주가 사실상 1회성** — 매 변주마다 렌더링 시간/리소스 비용이 발생
- **양산 한계** — 렌더 대기 시간 때문에 에셋 종 자체가 늘어나지 못함
- **접근성 부재** — 결과물이 Figma 안에만 존재, 실무자가 직접 가져다 쓰기 어려움

한편 사내 [3D 그래픽 에셋 사용 유형 조사](https://www.notion.so/3D-343a597878a080b39b1cf67efbab3a5c?pvs=21) 결과, 실제 수요는 **키비주얼급 고품질 객체보다 숏컷·아이콘 위계의 시각적 대체재**에 집중되어 있음 → 오프라인 렌더 품질을 끝까지 추구할 필요가 없는 영역.

---

## 2. 해결 방향 — WebGL/Three.js 실시간 렌더

- `MeshPhysicalMaterial`(PBR) + HDR 환경맵(`Botanical_Gardens_2_quarter.hdr`) + 투과/클리어코트 지원 → **아이콘·숏컷 사이즈에서 오프라인 렌더와 구분이 어려운 품질**
- 한 번 로드하면 **무한 앵글·무한 컬러 변주가 실시간** → 리소스 모델 자체를 재설계
- 콘텐츠/마케팅/CRM 등 **각 직무가 자기 문맥에 맞는 컷을 직접 생성**

> **Three.js 한 줄 요약**
> 브라우저에 내장된 저수준 3D 기능(WebGL) 위에서 동작하는 자바스크립트 라이브러리. "웹 브라우저 안에 작은 촬영장(Scene) 하나 차려놓고, 사용자가 카메라(Camera) 돌리고 재질(Material) 갈아끼워 원하는 컷을 뽑아가게 하는 시스템"이라고 보면 됨.

---

## 3. 프로젝트 요약

- **개발 체계** — 엔지니어 투입 없이 **브랜드팀이 Claude Code와 협업하여 자체 구축**
- **공급 형태** — 디자이너가 C4D에서 만든 에셋을 **glTF**로 내보내, 웹 브라우저의 Three.js 런타임에서 실시간 조작
- **사용 흐름** — ① 검색 → ② 카메라 앵글 조정 → ③ 브랜드 팔레트 내 컬러 선택 → ④ **1080×1080 투명 PNG 다운로드**
- **확장 방향** — 인터랙션 에셋 / 축적된 3D 데이터를 AI 이미지 생성의 학습·참조 소스로 활용

### 브랜드 일관성 제약

컬러 변주 가능 영역은 **그래픽 디자이너가 메쉬 단위로 사전 지정**. Three.js 런타임은 메쉬 이름 규칙을 읽어 다음과 같이 분류:

| 카테고리 | 동작 |
|---|---|
| `body` | 사용자가 선택한 Frosted 팔레트로 스왑 |
| `ribbon` | 디자이너 지정 액센트 컬러 고정 (예: 선물상자 리본) |
| `luminance` | 씬 조명 영향 받지 않는 순백 — 라벨/하이라이트용 |
| `glass` | 투명/투과 — 정수기 물탱크 등 |
| `chrome` | 금속 반사 — 손잡이/노즐 등 |

## 6. 제작·배포 파이프라인

1. **디자이너가 Cinema 4D에서 브랜드의 심상을 담은 3D 에셋을 제작**한다.
   단순한 형상 모델링이 아니라, [2025 Brand Graphic Design System](https://www.figma.com/design/JTmMbRsPUbHwslVOEj9qOl/2025-Brand-Graphic-Design-System?node-id=6756-19023&t=NcVXh5FGQP6qkz6Z-1)에서 정의 중인 **Authenticity(진정성) 조형 원칙**을 따라 형태·면·그라데이션을 설계한다.

   오늘의집의 3D 에셋은 **"이렇게 살아보고 싶다"는 전 세계 사람들의 꿈을 현실로 만든다**는 브랜드 가치를 시각적 심상으로 옮기는 작업이며, 다음 네 가지 원칙 위에서 제작된다.

   | 원칙 | 의미 | 시각적 표현 |
   |---|---|---|
   | **Smoothing Line** *(부드러움)* | 브랜드 심볼의 외형을 닮은 곡선으로 조형적 부드러움을 유지 | 모서리·면의 라운딩, 각진 단면 지양 |
   | **유저의 영감과 꿈** *(다정함)* | 브랜드의 다정한 정서와 꿈의 분위기 | 은은한 Vertical Gradient + 약간의 투명한 질감 |
   | **실현** *(명확성)* | 꿈을 실제로 보여준다는 가치를 보조하기 위해, 작은 화면에서도 또렷이 읽히는 형태 | 군더더기 없는 명확한 실루엣, 단순한 매스 |
   | **자세(Attitude)** *(안정감)* | 위 세 가지에 신뢰와 확신을 더하기 위한 단단하고 안정적인 무게감 — 캐주얼함 지양 | 지면에 또렷이 붙은 그림자(Ground Shadow) |

   네 원칙은 독립적으로 적용되는 항목이 아니라 **하나의 인상으로 합쳐지는 레이어**다. 부드러움 → 다정한 정서 → 명확한 실현 → 그것을 받쳐주는 안정된 자세, 의 순서로 읽힌다.

   동시에 런타임이 컬러 변주 영역을 인식할 수 있도록, 메쉬에 규칙대로 이름을 부여한다 (`body`, `ribbon`, `glass`, `chrome`, `luminance`).

2. **glTF(.gltf / .glb)** 포맷으로 익스포트 — 업계 표준, Three.js 네이티브 지원.

3. 라이브러리 리포지토리에 에셋 파일을 추가하고, 매니페스트에 등록한다.

4. **Three.js 런타임**이 에셋을 로드 → HDR 환경맵 기반 PBR 셰이딩 적용 → 사용자에게 실시간 뷰를 제공.

5. 사용자는 앵글·컬러를 확정한 뒤 **1080×1080 투명 PNG**로 내보낸다.

### 컬러 시스템 (현재 등록)

- **Frosted 7종 (Vertical Gradient, top→bottom)** — Blue, Green, Orange, Red, Violet, Dark Blue, Pink
- **빌트인 고정** — Luminance White, Ribbon Gray, Glass, Chrome
- 추후 브랜드팀 리뷰를 통해 확장
