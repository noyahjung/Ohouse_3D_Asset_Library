# 3D 에셋 라이브러리 개발 개요

OKR2의 두 KR을 함께 실행하는 프로젝트입니다.

- **KR2. 표준 시각 자산 라이브러리 개발** — 실행·적용 단계
- **KR3. AI 기반 아이콘 생성 파이프라인 설계** — 확장 단계

전 영역에 즉시 적용 가능한 표준 시각 자산을 공급하면서, 동시에 AI 기반 업무 파이프라인의 실증 사례로 삼는다.

---

## 프로젝트 요약

- **개발 체계**: 개발 인력 투입 없이, 클로드 코드(Claude Code)의 도움으로 브랜드팀이 자체 구축
- **제공 형태**: 그래픽 디자이너가 만든 3D 에셋을 glTF로 내보내어, 웹 브라우저의 Three.js 런타임에서 실시간으로 조작 가능한 형태로 제공
- **핵심 사용 흐름**: 사용자가 웹에서
  1. 에셋 검색
  2. 카메라 앵글 조정
  3. 지정된 브랜드 팔레트 내에서 컬러 선택
  4. PNG 다운로드 후 각자의 과업에 즉시 활용
- **확장 방향**: 인터랙션 에셋으로 확장, 축적된 3D 데이터를 AI 이미지 생성의 학습/참조 소스로 활용

---

## 프로젝트 배경

### 현황 진단

- 오늘의집의 3D 에셋은 아직 스타일 정의·고도화 단계 → 데이터 축적 이후 AI 기반 이미지 생성 자동화로 확장 가능
- 사내 3D 그래픽 에셋의 실제 용처 조사 결과, **키비주얼용 고퀄리티 객체보다 숏컷·아이콘 위계의 시각적 대체재**로서의 수요가 훨씬 많음

### 기술 선택의 타당성

- WebGL / Three.js 렌더링은 `MeshPhysicalMaterial`(PBR), HDR 환경맵, 투과·클리어코트 등을 지원해 **아이콘·숏컷 사이즈에서는 오프라인 렌더와 구분이 어려운 품질**을 즉시 확보
- 오프라인 렌더 파이프라인이 앵글별로 분·시간 단위 렌더를 반복 발주해야 하는 반면, WebGL 방식은 **무한 앵글·무한 컬러 변주를 실시간 제공** → 리소스를 근본적으로 재설계
- 다양한 직무(콘텐츠 / 마케팅 / CRM 등)가 각자의 문맥에 맞는 앵글·컬러 조합을 직접 생성·활용 가능 → 활용성과 확장성이 높음

## 서비스 구조 (로드맵 기준)

### Page 1 — 에셋 브라우저 *(예정)*

리스트 또는 팔레트 형태로 에셋을 검색·선택.

### Page 2 — 에셋 커스터마이저 *(현재 프로토타입 단계)*

선택한 에셋의 카메라 뷰·컬러를 조정하고 PNG로 다운로드.

### 브랜드 일관성 제약

각 에셋의 컬러 변주 가능 영역은 **그래픽 디자이너가 메쉬 단위로 사전 지정**한다.
예: 선물상자의 리본은 white 고정, 박스 본체만 변동 허용.

구현상으로는 메쉬 이름 규칙에 따라 `body / luminance / glass / chrome` 등으로 분류해 팔레트 스왑 허용 여부를 결정한다.

---

## 에셋 제작 가이드

브랜드의 Authenticity(진정성)를 시각화하는 조형 원칙:

- **Smoothing Line** — 조형적 부드러움을 유지
- **형태적 명확성** — 과하게 복잡하지 않고 시인성 높은 심플한 형태
- **은은한 그라데이션** — 단색이 아닌 Vertical Gradient로 브랜드 심상 표현
- **무게감의 유지** — 지나치게 가볍거나 캐주얼한 인상 지양

---

## 제작·배포 파이프라인

1. **그래픽 디자이너**가 Cinema 4D에서 브랜드 원칙에 맞는 3D 에셋을 제작
2. **glTF(.gltf / .glb)** 포맷으로 익스포트 *(업계 표준, Three.js 네이티브 지원)*
3. 라이브러리 리포지토리에 에셋 추가 및 메쉬 분류 규칙 정의 (어떤 파트가 컬러 변주 가능한지)
4. **Three.js 런타임**이 에셋을 로드 → HDR 환경맵 기반 PBR 셰이딩 적용 → 사용자에게 실시간 뷰 제공
5. 사용자는 앵글·컬러 지정 후 **1080 × 1080 투명 PNG**로 내보내기

### 컬러 시스템

3D 에셋 전용으로 튜닝된 브랜드 팔레트.

- 현재: Frosted Blue, Frosted Green + Glass / Chrome / White (빌트인)
- 추후 브랜드 팀 리뷰를 통해 확장
